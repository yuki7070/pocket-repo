# Pocket Repo 画面設計

## 方針

画面構成はGitHub Webのリポジトリコード閲覧画面に近づける。スマホから片手でリポジトリの状態、ファイル、Markdown、Diff、検索結果を確認できることを優先する。

MVPでは作成・編集・削除・Git操作を行うUIは置かない。

## 画面一覧

* リポジトリ選択画面
* リポジトリトップ画面
* ファイルブラウザ
* ファイルビューア
* Markdownプレビュー
* Git Status画面
* Diff Viewer
* 検索結果画面
* 設定画面

## ナビゲーション

### モバイル

* 上部にリポジトリ名と現在のbranch / worktreeを表示する
* 下部または上部に主要タブを置く
* 主要タブは `Code`、`Status`、`Search`、`Settings` を基本とする
* ファイル階層はパンくずリストで戻れるようにする

### デスクトップ

* GitHub Webに近い横幅のあるレイアウトにする
* ファイル一覧と本文表示の余白を広げる
* モバイルと同じ情報構造を維持する

## リポジトリ選択画面

目的

* 最近開いたリポジトリへ戻る
* 新しいリポジトリパスを開く

表示

* 最近開いたリポジトリ
* リポジトリ名
* パス
* Dirty状態
* 最終閲覧日時
* リポジトリパス入力

操作

* リポジトリを開く
* リポジトリパスを入力して開く

## リポジトリトップ画面

目的

* リポジトリの現在状態を把握する
* ファイル閲覧に入る

表示

* リポジトリ名
* 現在ブランチ
* worktree切り替え
* Dirty状態
* ファイル一覧
* READMEプレビュー

GitHub WebのCodeタブに近い構成にする。

## ファイルブラウザ

目的

* ディレクトリを移動する
* ファイルを選択して閲覧する

表示

* パンくずリスト
* ディレクトリ一覧
* ファイル一覧
* ファイル種別アイコン
* Git statusバッジ

操作

* ディレクトリへ移動
* ファイルを開く
* パンくずで上位階層へ戻る

## ファイルビューア

目的

* コードやテキストファイルを読む

表示

* ファイル名
* パス
* 行番号
* シンタックスハイライト
* 横スクロール

対応

* TypeScript
* JavaScript
* JSON
* YAML
* Markdown
* Python
* Shell

## Markdownプレビュー

目的

* READMEやdocsをスマホで読みやすく確認する

表示

* GitHub互換に近いMarkdown表示
* 見出しジャンプ
* コードブロックのシンタックスハイライト
* ダークモード

## Git Status画面

目的

* 未コミット変更の全体像を確認する

表示

* Modified
* Added
* Deleted
* Renamed
* Untracked
* staged / unstagedの区別

操作

* ファイルを選択してDiff Viewerへ移動する

## Diff Viewer

目的

* 未コミット差分をスマホで確認する

表示

* ファイル単位の差分
* インライン差分
* 追加行、削除行のハイライト
* staged / unstagedの表示

操作

* 差分対象ファイルの切り替え
* Status画面へ戻る

## 検索結果画面

目的

* ripgrepによる全文検索結果を確認する

表示

* 検索クエリ
* ファイルパス
* 行番号
* 該当行プレビュー

操作

* 検索結果からファイルビューアへ移動する

## 設定画面

目的

* Pocket Repo自身の表示設定を変更する

設定候補

* テーマ
* コードフォントサイズ
* 行折り返し
* サーバー設定の表示

MVPでは設定画面からリポジトリ内ファイルやGit状態を変更しない。
